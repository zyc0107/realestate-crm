# 房产CRM系统更新指南

## 本次更新内容 (2026-03-02)

本次更新实施了7项功能优化,提升系统易用性和功能完整性:

### ✅ 1. 录入房源字段调整
- 删除"详细地址"和"所在楼层"字段,简化录入流程
- 将"单元/房号"拆分为"单元"和"房号"两个独立字段
- 自动生成地址: 小区名称+单元+房号
- 优化字段布局顺序,提升录入效率

### ✅ 2. 房源管理展示优化
- 调整列表展示顺序: 房源信息 → 面积 → 价格 → 业主信息 → 状态 → 操作
- 新增独立的"面积"列,更清晰展示房源面积
- 房源信息列显示"小区名称 + 单元房号",删除详细地址显示
- 添加"详情"按钮和详情弹窗,方便查看完整房源信息

### ✅ 3. 客户预算价格显示优化
- 卖家客户显示关联房源的价格区间(最低价~挂牌价)
- 买家客户继续显示预算区间
- 更准确地反映客户的价格信息

### ✅ 4. 管理员归属信息展示
- 管理员视图下,房源和客户列表显示"所属门店"和"经纪人"列
- 方便管理员监控数据归属和经纪人业绩
- 普通中介不显示这些列

### ✅ 5. Excel导出功能验证
- 验证现有功能满足需求
- 支持Excel和CSV格式导出
- 管理员导出所有数据,中介导出自己的数据
- 包含房源、客户、交易三个工作表

### ✅ 6. 关联房源显示优化
- 客户管理页面的"关联房源"列显示完整的"小区+单元+房号"
- 替代原有的备注显示,更直观清晰

### ✅ 7. 回访提醒弹窗功能
- 到达提醒时间后自动弹出提醒窗口
- 每分钟自动检查一次提醒
- 支持标记完成功能
- 使用localStorage避免重复弹窗
- 美观的弹窗样式

## 手动部署步骤

### 方式一: SSH登录服务器部署 (推荐)

1. **SSH登录到腾讯云服务器**
   ```bash
   ssh root@182.254.146.23
   ```

2. **进入项目目录**
   ```bash
   cd /var/www/realestate-crm
   ```

3. **拉取最新代码**
   ```bash
   git pull origin main
   ```

4. **安装后端依赖** (如果有新依赖)
   ```bash
   cd backend
   npm install
   ```

5. **安装前端依赖** (如果有新依赖)
   ```bash
   cd ../frontend
   npm install
   ```

6. **构建前端**
   ```bash
   npm run build
   ```

7. **重启后端服务**
   ```bash
   cd ../backend
   pm2 restart realestate-backend
   ```

8. **查看服务状态**
   ```bash
   pm2 status
   pm2 logs realestate-backend --lines 20
   ```

9. **验证部署**
   - 访问: http://182.254.146.23
   - 使用管理员账号登录: admin / admin123
   - 测试新功能是否正常工作

### 方式二: 使用部署脚本 (自动化)

如果你的本地环境可以SSH到服务器,可以直接运行部署脚本:

```bash
cd /path/to/realestate-crm
bash deploy.sh
```

脚本会自动完成以下步骤:
1. 拉取最新代码
2. 安装依赖
3. 构建前端
4. 重启后端服务
5. 查看服务状态

## 数据兼容性说明

本次更新完全兼容旧数据:

- **房源数据**: 旧数据的 `address` 和 `unit_room` 保持不变,新录入数据自动生成地址
- **客户数据**: 如果客户没有关联房源,显示"-"
- **归属信息**: 如果 `created_by` 为NULL(旧数据),显示"-"

## 回滚方案

如果部署后发现问题,可以快速回滚到上一个版本:

```bash
# SSH登录服务器
ssh root@182.254.146.23

# 进入项目目录
cd /var/www/realestate-crm

# 回滚到上一个版本
git reset --hard HEAD~1

# 重新构建前端
cd frontend
npm run build

# 重启后端服务
cd ../backend
pm2 restart realestate-backend
```
